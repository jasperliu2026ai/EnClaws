# Code of Conduct

The EnClaws community aims to be a professional, measured, fact-respecting, and collaboration-friendly place.
We discuss systems, but we deal with people.

## 1. Our Commitment

Regardless of your background, experience, role, language, region, or viewpoint, if you participate in the project constructively, we want you to be able to discuss, contribute, and collaborate here safely.

## 2. Encouraged Behavior

- Respect other people's time, experience, and boundaries
- Focus on the issue itself and provide verifiable information and context
- Accept and offer constructive feedback
- Be careful with security, privacy, compliance, and multi-tenant boundaries
- In disagreement, address the technical point rather than turning it into a personal judgment
- Acknowledge mistakes and help fix problems when you discover them

## 3. Unacceptable Behavior

The following behavior is unacceptable in the community:

- Harassment, discrimination, threats, humiliation, insults, or personal attacks
- Malicious spam, trolling, baiting, flame wars, or sustained unwanted contact
- Publishing someone else's private information, sensitive logs, customer data, contact details, or security details without permission
- Repeatedly spreading information known to be false or materially misleading
- Using project channels for ad spam, scams, social engineering, or unrelated marketing
- Misleading others with claims such as "official announcement" or "official partnership"

## 4. Scope

This Code of Conduct applies to:

- GitHub Issues, Pull Requests, and Discussions
- Repository-related documentation, comments, and review records
- Public interactions where maintainers represent EnClaws
- Other online or offline spaces clearly designated as project community spaces

## 5. Enforcement

The maintainers may take action as appropriate, including but not limited to:

- A reminder or warning
- Editing or removing inappropriate content
- Locking a discussion
- Declining to merge or restricting collaboration access
- Temporarily or permanently limiting participation

Enforcement should aim to protect community safety and collaboration, not to create public humiliation.

## 6. Reporting

If you encounter behavior that violates this Code of Conduct, please contact the maintainers privately when possible.
If no private contact channel is currently public, you may open a minimal issue **without details** to request a private reporting channel.

## 7. Additional Notes

- Reasonable technical disagreement, strict code review, and evidence-based dissent do not by themselves violate this policy.
- But a technical disagreement is never a free pass for personal attacks.
