# 行为准则

EnClaws 社区致力于成为一个专业、理性、尊重事实、鼓励协作的空间。
我们讨论的是系统，但面对的是人。

## 1. 我们的承诺

无论你的背景、经验、角色、语言、地区或观点如何，只要你以建设性的方式参与项目，我们希望你能够在这里安全地讨论、贡献和协作。

## 2. 鼓励的行为

- 尊重他人的时间、经验和边界
- 聚焦问题本身，提供可验证的信息和上下文
- 接受并提供建设性的反馈
- 注意安全性、隐私、合规性和多租户边界
- 在分歧中，针对技术观点讨论，而非对人进行评判
- 承认错误，发现问题时帮助修复

## 3. 不可接受的行为

以下行为在社区中不可接受：

- 骚扰、歧视、威胁、羞辱、侮辱或人身攻击
- 恶意刷屏、挑衅、引战、持续的不受欢迎的联系
- 未经许可发布他人的私人信息、敏感日志、客户数据、联系方式或安全细节
- 反复传播明知虚假或具有重大误导性的信息
- 利用项目渠道发布广告垃圾信息、诈骗、社会工程攻击或无关的营销内容
- 以"官方公告"或"官方合作"等名义误导他人

## 4. 适用范围

本行为准则适用于：

- GitHub Issues、Pull Requests 和 Discussions
- 与仓库相关的文档、评论和审查记录
- 维护者代表 EnClaws 的公开互动
- 其他明确指定为项目社区空间的线上或线下场所

## 5. 执行

维护者可根据情况采取适当措施，包括但不限于：

- 提醒或警告
- 编辑或删除不当内容
- 锁定讨论
- 拒绝合并或限制协作权限
- 临时或永久限制参与

执行的目标是保护社区安全与协作，而非制造公开羞辱。

## 6. 举报

如果你遇到违反本行为准则的行为，请尽可能私下联系维护者。
如果目前没有公开的私下联系渠道，你可以提交一个**不包含详细信息**的 Issue，请求建立私下举报渠道。

## 7. 补充说明

- 合理的技术分歧、严格的代码审查和基于证据的异议本身不构成违规。
- 但技术分歧绝不是人身攻击的通行证。
